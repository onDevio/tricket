External Alexis Maeso Serrano commented a month ago on 09/05//2017 Spent: 0 Hours
Se celebra una reunión el 08/05/2017 para concretar los cambios que son necesarios para poder mantener contralados y actualizados los datos del SRC: - - - - 1. **Crear un nuevo campo de observaciones relacionadas con el SRC en la ficha del colegiado de SICAV.** 2. ** Crear automatismo independiente para pasar a No Ejerciente un colegiado que empiece a cumplir una sanción.** - No se ejecutará ninguna de las acciones automáticas que se ejecutan habitualmente con el paso a No Ejerciente de un colegiado. Si bien se guardará un registro asociado a la sanción del estado en el que se encontraba el colegiado en el momento de empezar a cumplir la sanción (Ejerciente o No Ejerciente). - Revisar y actualizar si resulta necesario el proceso de paso a Ejerciente para que en caso de estar cumpliendo sanción el colegiado, el usuario deba confirmar el paso a Ejerciente. - Los tipos de sanción a tener en cuenta son: - Sanción Deontológica - Sanción CGAE - Sanción Tribunales - Sanción Otros Colegios 3. **Crear automatismo independiente para pasar a Ejerciente un colegiado que finalice un periodo de sanción.** - Se revisará el registro de estado asociado a la sanción. - Si el registro indicaba que No era Ejerciente, no se realizará ninguna acción. - Si era Ejerciente, se pasará el colegiado a Ejerciente sin ejecutar ninguna de las acciones que se ejecutan habitualmente con el paso a Ejerciente de un colegiado. 4. **Detectar colegiados que se dan de baja o paso a no Ejerciente de SICAV,** si consta que el colegiado es socio de una de las sociedades registradas se deberá mostrar un mensaje en pantalla que indique al usuario que se debe solicitar el acta notarial de la modificación a la sociedad. 5. **Detectar en el proceso de reincorporación de colegiados o paso a Ejerciente de SICAV,** si consta que el colegiado es socio de una de las sociedades registradas se deberá mostrar un mensaje en pantalla que indique al usuario que se debe solicitar el acta notarial de la modificación y si queda incluido en el SRC de la compañía, la documentación que lo acredite. 6. **Modificar el registro de sociedades en SICAV** para: - Crear un nuevo permiso que permita eliminar socios de tipo colegiado cuando el colegiado en cuestión no dispone de SRC a nivel individual. - Si se pretende dar de baja como socio a un colegiado del ICAV: - Se mostrará un aviso al usuario para recordarle que debe solicitar el acta notarial de la modificación. - Si el colegiado en cuestión no dispone de SRC a nivel de persona física: - Si el usuario no dispone de permisos, se indicará que la baja la deberá tramitar el supervisor de administración. - Si el usuario dispone de permisos, se le preguntará si el colegiado dispone de SRC con alguna compañía o con el colegio. - Dispone de SRC con otra compañía: se guarda los datos y se abre la ficha del colegiado para introducir los datos. - Dispone de SRC con el colegio: Se inserta registro con los datos correspondientes. - ¿Deberíamos dar la opción de indicar que no dispone de SRC? - Si se da de alta un nuevo colegiado en una Sociedad se recordará al usuario que debe solicitar el acta de alta o modificación, según corresponda, de la sociedad. - Si está incluido dentro del SRC de la sociedad se deberá solicitar la documentación acreditativa. - Crear un listado para controlar los datos de SRC. El listado indicará: - Razón social - CIF - Dispone de SRC. SI / NO - Compañía SRC - Fecha Vencimiento - Domicilio activo Fiscal - Lista socios colegiados del ICAV - Nº colegiado - Nombre y Apellidos - Dirección de correspondencia o principal en su defecto. 7. **Modificar proceso de Baja de colegiados para que el SRC pase a No Disponible.** 8. **Crear sistema independiente de avisos automáticos para controlar el vencimiento del SRC de otras compañías.** - Se permitirá configurar: - El día de aviso del mes para avisar. Indicar inicialmente el día 25. - Tiempo de antelación del aviso. Por defecto 2 meses - El correo al cual avisar. Inicialmente adminitracion@icav.es - El aviso contendrá: - Listado de colegiados Ejercientes de alta con SRC de otras compañías que vence en el plazo configurado. - Listado de Sociedades que no estén de baja cuyo SRC vence en el plazo configurado. - Además de realizar un aviso a la dirección de correo configurada, se enviará otro correo personalizado al colegiado interesado o a los colegiados del ICAV que forman parte de la sociedad registrada. (Confirmar el texto de los correos) - El listado de colegiados dispondrá de: - Nº de colegiado - Nombre y Apellidos - Dirección fiscal o en su defecto, dirección principal. - Compañía SRC - Fecha de Vencimiento - El listado de sociedades dispondrá de: - Razón social - CIF - Compañía SRC - Fecha Vencimiento - Domicilio activo Fiscal - Lista socios colegiados del ICAV - Nº colegiado - Nombre y Apellidos - Dirección de correspondencia o principal en su defecto. 9. **Crear sistema independiente de actualización automático de SRC de los colegiados.** - El sistema se ejecutará diariamente - El sistema deberá actualizar automáticamente la ficha colegial para indicar que dispone de SRC colegial, guardando las correspondientes entradas en el historial, en caso de que se encuentre: - Un colegiado Ejerciente sin datos de SRC, siempre y cuando no pertenezca a una sociedad con SRC en vigor. - Un colegiado Ejerciente cuyo SRC con otra compañía haya vencido, independientemente de si está relacionado con una sociedad. - Se enviará un listado de todos los colegiados actualizados al correo configurado. Por defecto, administracion@icav.es - **No se facturará el SRC.** 10. **Crea sistema independiente de actualización automático de SRC de las Sociedades.** - El sistema se ejecutará diariamente - El sistema deberá actualizar automáticamente la ficha de la sociedad para indicar que No Consta SRC en caso de vencimiento de la póliza actual. - Se enviará un listado de todas las sociedades actualizadas al correo configurado.
